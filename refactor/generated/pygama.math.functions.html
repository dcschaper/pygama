pygama.math.functions.crystall_ball module

pygama.math.functions.crystall_ball.nb_xtalball_cdf(x: float, beta: float, m: float, mu: float, sigma: float, A: float) → float

CDF for power-law tail plus gaussian. As a Numba vectorized function, it runs slightly faster than 'out of the box' functions. It computes

\[\begin{split}y = \frac{x-\mu}{\sigma} \\ PDF = A \times cdf(y, \beta, m)\end{split}\]

Note

TODO:: potentially replace by a guvectorized factory function so that the normalization isn't computed every time

Parameters

  • x (array-like) -- The input data

  • beta (float) -- The point where the Cdf changes from power-law to Gaussian

  • m (float) -- The power of the power-law tail

  • mu (float) -- The amount to shift the distribution

  • sigma (float) -- The amount to scale the distribution

  • A (float) -- An overall scale factor

Returns

y (float) -- The value at x given the parameters

Return type

float

pygama.math.functions.crystall_ball.nb_xtalball_pdf(x: float, beta: float, m: float, mu: float, sigma: float, A: float) → float

PDF of a power-law tail plus gaussian. As a Numba vectorized function, it runs slightly faster than 'out of the box' functions. It computes

\[\begin{split}y = \frac{x - \mu}{\sigma} \\ PDF = A \frac{pdf(y, \beta, m)}{\sigma}\end{split}\]

Note

Later: potentially replace by a guvectorized factory function so that the normalization isn't computed every time

Parameters

  • x (array-like) -- The input data

  • beta (float) -- The point where the pdf changes from power-law to Gaussian

  • m (float) -- The power of the power-law tail

  • mu (float) -- The amount to shift the distribution

  • sigma (float) -- The amount to scale the distribution

  • A (float) -- An overall scale factor

Returns

y (float) -- The value at x given the parameters

Return type

float
